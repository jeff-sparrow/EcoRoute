
## [x] [#3: Inaccessible Route Timeout](/../../issues/3)
When routing to an inaccessible (e.g. Eugene -> Berlin) or very far destination, EcoRoute timed out, not showing UI elements aside from the map and directions button.

## [ ] [#4: Specific destinations do not show up in route search results](/../../issues/4)
When attempting to search for a destinations like addresses, they would not show up for user in selectable results.

## [x] [#5: Carbon emission estimates are obviously wrong](/../../issues/5)
When showing the available modes of transport for a route, walking is stated to produce more emissions than a car, and biking is stated to produce a similar amount of emissions.

## [ ] [#6: User is unable to set start coordinates](/../../issues/6)
While the user can share their location, and the program assumes that to be their chosen starting location, they are not able to enter specified start coordinates other than that.

## [x] [#12: Hamburger icon does not back user out of route display modal](/../../issues/12)
The hamburger icon in the route display modal does not back the user out of the route display modal, requiring a refresh to return to the base screen.

## [x] [#13: The green preference slider state does not persist](/../../issues/13)
When exiting route display or refreshing, the green preference slider (faster greener slider)'s state does not persist.

## [ ] [#14: Sign in functionality](/../../issues/14)
Beta testers found that the sign in function did not exist.

## [ ] [#15: Smaller/resized screen causes elements to overlap](/../../issues/15)
When on a smaller screen or having resized the window, elements such as the search bar, green preference slider, hamburger icon and sign in button overlap.

## [ ] [#16: Certain menu buttons are completely unresponsive](/../../issues/16)
The hamburger icons (in the route display modal and main screen) and sign in button were found to be unresponsive.

## [ ] [#17: Location defaults to UO when location is not provided](/../../issues/17)
When EcoRoute is unable to resolve the user's location, it defaults to UO.
